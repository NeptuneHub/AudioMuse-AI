Measured against that server's library size as counted by its last alignment sweep, so it is the one number that tells you whether analysis still has work to do. It is per server on purpose: a track that has never been analyzed exists on no catalogue, so there is no catalogue-wide version of this figure.

Music Server Status Per server Each server holds its own library; the catalogue is the union of all of them, deduplicated. A song present on two servers is ONE song in the catalogue, so these per-server counts add up to more than the catalogue total - never sum them. "Unique in catalogue" counts THAT server's distinct songs matched to the catalogue; "Duplicate files" are extra copies of a song already counted (same audio, several files). "Coverage" is the provider tracks resolved to the catalogue, measured against the server's own library size from its last alignment sweep.
