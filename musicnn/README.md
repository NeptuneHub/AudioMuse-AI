# MusiCNN → ONNX Export & Embedding/Prediction Comparison

Export the **MSD-MusiCNN embedding model** and **classification head** from
[jordipons/musicnn](https://github.com/jordipons/musicnn) to ONNX format
and compare against the original TF model and the project's existing
`msd-musicnn-1.onnx` (embedding) and `msd-msd-musicnn-1.onnx` (prediction).

Two ONNX models are produced:

| Model | Input | Output | Size |
|---|---|---|---|
| `musicnn_embedding.onnx` | mel `(batch,187,96)` | embedding `(batch,200)` | ~3 MB |
| `musicnn_prediction.onnx` | embedding `(batch,200)` | logits `(batch,50)` | ~43 KB |

---

## Setup

### 1. Create a virtual environment

```bash
cd musicnn
python3 -m venv .venv
```

### 2. Activate it

```bash
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
pip install --no-deps musicnn
```

> **Why two commands?** The `musicnn` PyPI package pins `numpy<1.20` which
> cannot build on Python 3.11+ / Apple Silicon. Installing with `--no-deps`
> skips that broken constraint while still getting the library and its
> bundled model weights.

## Usage

### Step 1 — Export the model to ONNX

```bash
python export_onnx.py
```

This loads the MSD-MusiCNN TensorFlow checkpoint bundled with the `musicnn`
library, freezes the graph, removes the `is_training` flag, and saves
`musicnn_embedding.onnx` in this directory.

**ONNX model spec:**

| | Name | Shape | Type |
|---|---|---|---|
| Input | `model/Placeholder:0` | `(batch, 187, 96)` | float32 |
| Output | `model/dense/BiasAdd:0` | `(batch, 200)` | float32 |

### Step 2 — Compare embeddings

```bash
python compare_embeddings.py
```

Processes every audio file in `../test/songs/` through three models:

| # | Source | Description |
|---|---|---|
| 1 | **musicnn TF** | Original TF session from the `musicnn` library |
| 2 | **Our ONNX** | `musicnn_embedding.onnx` (exported in Step 1) |
| 3 | **Project ONNX** | `../test/models/msd-musicnn-1.onnx` |

For each song the script reports:

- **Pairwise cosine similarity** (and cosine distance = 1 − cos)
- **Max absolute difference** between per-patch embeddings
- **Inference time** for each model

A final summary table shows all results. The pass threshold is
cosine similarity ≥ 0.99 for every pair.

### Step 3 — Export the classification head to ONNX

```bash
python export_prediction_onnx.py
```

Extracts the `batch_normalization_10` and `dense_1` weights from the TF
checkpoint and builds a lightweight standalone ONNX model:

```
embedding (batch,200) → ReLU → BatchNorm → MatMul + BiasAdd → logits (batch,50)
```

**ONNX model spec:**

| | Name | Shape | Type |
|---|---|---|---|
| Input | `serving_default_model_Placeholder:0` | `(batch, 200)` | float32 |
| Output | `PartitionedCall:0` | `(batch, 50)` | float32 |

The script also verifies the export against the TF reference (`max_diff < 1e-4`)
and compares predictions (after sigmoid) against the project's
`msd-msd-musicnn-1.onnx` on the test songs.

> **Note:** The output is raw logits; apply `sigmoid()` to get probabilities.
> The 50 output dimensions correspond to the MSD top-50 tags.
